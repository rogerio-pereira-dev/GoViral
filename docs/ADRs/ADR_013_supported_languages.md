# ADR-013: Supported Report Languages

## Status

Approved

## Context

The target audience is global (beginner and small creators). The report generated by the LLM must be delivered in the language chosen by the user for clarity and adoption.

## Decision

Support **three languages** in the MVP:

- **en** — English
- **es** — Spanish (Español)
- **pt** — Portuguese (Português)

The language is selected on the landing page, stored in the `locale` field of the analysis request, and passed to the LLM (e.g. placeholder `{{LANGUAGE}}` in the prompt template) so that all report content is generated in the corresponding language.

## Consequences

- **Positive:** Better experience in EN/ES/PT markets; explicit choice on the form avoids ambiguity for the LLM.
- **Negative:** Prompt and tests must cover all three languages; possible quality variation across languages depending on the model used.
- **Neutral:** Adding new languages in the future is possible via a new `locale` value and prompt/UI updates.
