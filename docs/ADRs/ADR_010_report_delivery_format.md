# ADR-010: Report Delivery Format

## Status

Approved

## Context

The analysis result must be delivered to the user in a consumable form. Options include PDF, HTML attachment, HTML in the email body, temporary download link, or dashboard.

## Decision

Deliver the report **exclusively by email**, in the **message body as HTML**. Do not generate PDF in the MVP, do not store the report for later access, and do not offer a download link.

The email is sent via AWS SES after the HTML is generated by the pipeline (LLM + template).

## Consequences

- **Positive:** Simple implementation (Laravel Mail + HTML view); no file storage; direct delivery to the inbox; aligned with the “no history” policy.
- **Negative:** User may lose the email; limited to clients that render HTML correctly; no “second copy” without a new purchase.
- **Neutral:** Post-MVP evolution (e.g. PDF, temporary link) possible via a new decision.
